# DNX Store V8.8 — исправление характеристик, модалки и TGS

## Что исправлено

1. Каталог больше не берётся из шестичасового снимка `localStorage`.
2. При открытии NFT приложение запрашивает свежую строку `items` по точному `id` через `GET /item`.
3. Значения характеристик берутся только из столбцов:
   - `items.model`
   - `items.pattern`
   - `items.background`
4. Старая кнопка «Продать» больше не может появиться в модалке каталога и ломать кнопки покупки.
5. В каталоге остаются только две кнопки: широкая «Купить» и квадратная корзина.
6. У TGS сохраняется первый видимый кадр. После окончания анимации NFT больше не должен превращаться в пустой цветной квадрат.
7. Качество TGS стало адаптивным: выше на мощных устройствах, а на слабых Android ограничиваются параллельные загрузки и количество workers.
8. В кнопках продажи цена снова показывается со звездой одинакового с цифрами размера.

## Как установить

1. Откройте GitHub-репозиторий `dnx-store`.
2. Нажмите **Add file → Upload files**.
3. Загрузите всё содержимое папки `dnx-store-stars-final-v8.8` с заменой существующих файлов.
4. Особенно важно одновременно заменить три файла:
   - `index.html`
   - `main.py`
   - `tgsticker.js`
5. Нажмите **Commit changes**.
6. Откройте Render → нужный Web Service → **Deploys**.
7. Нажмите **Deploy latest commit**. Если включён Auto-Deploy, дождитесь статуса **Live**.
8. После деплоя полностью закройте мини-приложение Telegram и откройте снова. Старый локальный снимок каталога V8.7 будет удалён автоматически.

## Почему могло показываться старое значение вместо `kall`

В V8.8 кэш характеристик устранён. Если после одновременного деплоя `index.html` и `main.py` значение всё ещё не `kall`, это означает одно из двух:

1. В Neon изменена другая строка с таким же `name`, но другим `id`.
2. Render подключён другой строкой `DATABASE_URL` — например, к другой ветке Neon.

Проверьте в SQL Editor Neon:

```sql
SELECT current_database(), current_schema();

SELECT id, name, model, pattern, background, status, buyer_id
FROM items
WHERE name = 'Chill Flame'
ORDER BY id;
```

Менять нужно строку с тем `id`, на которую вы нажимаете. У товара в каталоге и экземпляра в инвентаре могут быть одинаковые названия, но разные `id` и разные характеристики.

Затем откройте Render → **Environment** и убедитесь, что `DATABASE_URL` ведёт в ту же базу и ту же Neon branch. Само значение пароля никому не отправляйте.

## Быстрая проверка

1. В `items` временно задайте для конкретного `id`: `model='kall'`, `pattern='kall'`, `background='kall'`.
2. Перезапустите мини-приложение и откройте именно этот товар.
3. В модалке все три строки должны показать `kall`.
4. Дождитесь окончания TGS: должна остаться видимая модель, а не пустой фон.
5. В каталоге проверьте, что кнопка покупки не переносится на несколько строк, корзина не перекрывает её, а кнопки «Продать» нет.
6. В инвентаре проверьте вертикальные кнопки «Вывести» и «Продать».
