# DNX Store V8.2 — установка и управление ограничениями

## 1. Какие файлы загружать

Из папки `dnx-store-stars-final-v8.2` замените в репозитории файлы:

- `main.py`
- `index.html`

Остальные изображения и служебные файлы из папки также должны остаться в репозитории. Новые переменные Environment не требуются.

## 2. Загрузка через GitHub

1. Откройте репозиторий DNX Store на GitHub.
2. Нажмите **Add file → Upload files**.
3. Перетащите содержимое папки `dnx-store-stars-final-v8.2`.
4. Подтвердите замену существующих `main.py` и `index.html`.
5. Внизу страницы укажите сообщение, например `DNX Store V8.2`.
6. Нажмите **Commit changes**.

Если Render подключён к этой ветке, новый деплой начнётся автоматически. В Render откройте сервис → **Events** и дождитесь статуса **Live**.

## 3. Автоматическое обновление базы

При первом запуске V8.2 сервер сам добавит:

- `users.mut` — целое количество часов ограничения;
- `users.ban` — бессрочная блокировка (`TRUE`, `FALSE` или `NULL`);
- `users.mut_until` — техническое точное время окончания ограничения;
- таблицу `user_events` для полной истории аккаунта.

Ручная миграция не нужна. Не удаляйте `mut_until`: он нужен для точного обратного отсчёта.

## 4. Как выдать временное ограничение

1. Откройте Neon.
2. Выберите проект и нажмите **SQL Editor**.
3. Вставьте запрос, заменив ID и количество часов:

```sql
UPDATE users
SET mut = 12
WHERE id = 'TELEGRAM_USER_ID';
```

4. Нажмите **Run**.

Таймер запускается в момент выполнения запроса. После окончания сервер автоматически заменит `mut` и `mut_until` на `NULL`.

Снять ограничение досрочно:

```sql
UPDATE users
SET mut = NULL
WHERE id = 'TELEGRAM_USER_ID';
```

## 5. Как заблокировать или разблокировать аккаунт

Заблокировать без ограничения по времени:

```sql
UPDATE users
SET ban = TRUE
WHERE id = 'TELEGRAM_USER_ID';
```

Разблокировать:

```sql
UPDATE users
SET ban = FALSE
WHERE id = 'TELEGRAM_USER_ID';
```

Можно вернуть исходное пустое значение:

```sql
UPDATE users
SET ban = NULL
WHERE id = 'TELEGRAM_USER_ID';
```

## 6. Что видит ограниченный пользователь

- При `mut` открывается незакрываемый экран с живым таймером.
- При `ban = TRUE` открывается незакрываемый экран бессрочной блокировки.
- Навигация, баланс, NFT и кнопки становятся недоступны.
- Сервер возвращает `423 Locked` на защищённые запросы, поэтому ограничение нельзя обойти прямым вызовом API.
- На экране есть ссылка на `@DnxStoreSupport`.

## 7. Полная история аккаунта

V8.2 сохраняет в `user_events` новые события после развёртывания версии:

- Telegram Stars и TON-пополнения;
- реферальные начисления;
- покупки NFT;
- ставки, возвраты, выигрыши и проигрыши;
- стоимость открытия кейса и выпавший приз;
- продажу кейсового NFT;
- запрос, подтверждение и отклонение вывода.

События не удаляются при продаже или выводе NFT. В приложении они показываются во вкладке **Инвентарь → История**, по 80 записей с кнопкой загрузки следующих.

## 8. Проверка после деплоя

1. Откройте мини-приложение обычным аккаунтом.
2. Проверьте игру, магазин и инвентарь.
3. Поставьте тестовому аккаунту `mut = 1` и перезапустите мини-приложение — должен появиться таймер.
4. Снимите `mut`, затем поставьте `ban = TRUE` — должен появиться бессрочный экран.
5. Верните `ban = FALSE`.
6. Сделайте небольшую тестовую ставку или покупку и проверьте **Инвентарь → История**.
7. Откройте таблицу лидеров на телефоне и компьютере: она должна занимать весь экран, показывать только три призовых места и отдельную строку пользователя, если он ниже третьего места.
