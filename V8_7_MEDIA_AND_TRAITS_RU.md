# DNX Store V8.7 — анимации NFT, кейсы и характеристики

## Что исправлено

1. Размер модели в анимированной NFT-области увеличен до 70%, фон занимает примерно 30%.
2. Для NFT с заполненным `nft_link` полностью исключена подмена статичной картинкой.
3. До запуска анимации показывается настоящий нулевой кадр TGS. При медленном интернете область остаётся аккуратной и не меняет размеры.
4. На компьютере анимация запускается на один цикл при наведении.
5. На телефоне анимация запускается на один цикл при появлении NFT на экране и снова запускается после ухода и возврата в видимую область.
6. В кейсе тап по анимированному предмету повторно запускает один цикл.
7. Статичные картинки анимированных предметов больше не предзагружаются в фоне.
8. Проценты возможных призов в кейсе выровнены и оформлены отдельными премиальными плашками.
9. Кнопки продажи выровнены, получили одинаковую высоту, скругление и корректное состояние блокировки.
10. Каталог получает характеристики только из `items.model`, `items.pattern`, `items.background`. Старый JSON `traits` больше не копируется и не используется каталогом.
11. Старый браузерный снимок каталога отключён. После обновления клиент сразу запрашивает актуальные данные из Neon.

## Как должны называться столбцы в Neon

В таблице `items` нужны ровно эти три столбца в нижнем регистре:

```text
model
pattern
background
```

Рекомендуемый тип каждого столбца — `varchar(255)` или `text`.

Если значение не задано, можно оставить `NULL` или пустую строку. В окне предмета Mini App покажет `None`.

Старый столбец `traits` для каталога больше не нужен. Создавать его снова не надо.

## Как проверить значения в Neon

1. Откройте сайт Neon и войдите в проект базы данных.
2. Слева откройте `SQL Editor`.
3. Вставьте запрос:

```sql
SELECT id, name, model, pattern, background, nft_link
FROM items
ORDER BY id;
```

4. Нажмите `Run`.
5. Найдите нужный товар и убедитесь, что данные находятся именно в трёх новых столбцах.
6. Если надо изменить характеристики, выполните, например:

```sql
UPDATE items
SET model = 'Kawaii Dog',
    pattern = 'Mannaz Rune',
    background = 'Strawberry'
WHERE id = 123;
```

Замените `123` на настоящий `id` товара.

## Как загрузить V8.7 на GitHub

1. Распакуйте архив `dnx-store-stars-final-v8.7.zip`.
2. Откройте репозиторий `denixl-11/dnx-store` на GitHub.
3. Нажмите `Add file` → `Upload files`.
4. Перетащите все файлы из распакованной папки. Перетаскивать надо содержимое папки, а не саму папку целиком.
5. Убедитесь, что заменяются как минимум `index.html`, `main.py` и `README.md`.
6. Добавьте также `V8_7_MEDIA_AND_TRAITS_RU.md`.
7. В поле описания напишите `DNX Store V8.7`.
8. Нажмите `Commit changes`.

## Как применить на Render

1. Откройте Render.
2. Откройте Web Service вашего бота.
3. Перейдите в `Manual Deploy`.
4. Нажмите `Deploy latest commit`.
5. Дождитесь статуса `Live`.
6. В логах должен появиться успешный запуск без Python traceback.
7. Переменные в разделе `Environment` менять не требуется.
8. Миграцию базы вручную запускать не требуется: существующие `model`, `pattern`, `background` используются напрямую.

## Как очистить старую версию в Telegram

После появления статуса `Live`:

1. Полностью закройте Mini App.
2. Закройте чат с ботом и заново откройте его.
3. Запустите Mini App снова.
4. Если Telegram продолжает показывать старый интерфейс, откройте ссылку Mini App один раз с добавленным параметром, например `?v=87`, либо очистите кеш Telegram WebView.

Ключ каталога в V8.7 уже изменён на `dnx_catalog_v3`, поэтому характеристики из старого локального снимка не должны возвращаться.

## Финальная проверка

1. Откройте каталог на телефоне: NFT должна один раз проиграться при появлении на экране.
2. Пролистайте её за пределы экрана и верните: должен проиграться ещё один цикл.
3. На компьютере наведите мышь: должен проиграться один цикл.
4. При медленной загрузке не должна появляться старая PNG-картинка другого размера.
5. Откройте корзину, инвентарь и окно характеристик — масштаб модели должен быть одинаковым.
6. Откройте кейс и тапните по анимированному призу — должен проиграться один цикл.
7. Проверьте, что плашки процентов находятся по центру верхней границы карточек и не съезжают.
8. Проверьте обычную и заблокированную кнопки продажи.
9. Измените `model`, `pattern` или `background` в Neon, заново откройте каталог и убедитесь, что показано новое значение.

## Важно

Статичная картинка всё ещё используется у обычных предметов без `nft_link`. Это ожидаемо: у таких предметов нет Telegram TGS-анимации и нулевого кадра, который можно отрисовать.
