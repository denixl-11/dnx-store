# DNX Store: очень подробное обновление на TON

Ниже описан безопасный порядок действий без командной строки. Сначала прочитайте инструкцию целиком, затем выполняйте по порядку.

## 0. Важное ограничение Telegram

Telegram официально требует использовать Telegram Stars для продажи цифровых товаров и услуг внутри ботов и Mini Apps. TON-платежи за цифровые NFT могут привести к ограничениям приложения со стороны Telegram. Эта сборка технически поддерживает TON Connect и автоматическое зачисление, но не отменяет правила платформы.

## 1. Что изменилось

1. Весь видимый знак рубля заменён на TON.
2. Пополнение происходит через официальный стандарт TON Connect: пользователь сам подтверждает перевод в подключённом кошельке.
3. Сервер не верит ответу браузера. Он отдельно получает транзакции из TON Center, проверяет адрес отправителя, точную сумму, время, уникальный hash и только после этого одним атомарным запросом увеличивает баланс.
4. Seed-фраза и приватный ключ кошелька администратора нигде не используются.
5. Ручное подтверждение пополнений администратором удалено. Вывод NFT остаётся прежним.
6. Цены, баланс, кейсы и ставки поддерживают дробные значения TON до 9 знаков.

## 2. Сделайте резервную копию

1. Откройте `https://github.com/denixl-11/dnx-store`.
2. Нажмите зелёную кнопку **Code**.
3. Нажмите **Download ZIP** и сохраните архив в отдельную папку.
4. Откройте Neon Console, выберите production-проект и убедитесь, что это та база, которую использует Render.
5. В Neon откройте **SQL Editor** и не выполняйте никаких запросов до шага 8.

## 3. Подготовьте кошелёк администратора

1. В Telegram откройте официальный кошелёк, которым вы будете принимать TON, либо другой кошелёк с поддержкой TON Mainnet.
2. Нажмите **Receive / Получить** → **Toncoin**.
3. Скопируйте публичный адрес формата `UQ...`, `EQ...` или `0:...`.
4. Отправьте на него маленький тестовый перевод и убедитесь, что кошелёк активен.
5. Никому не сообщайте seed-фразу. Не добавляйте seed-фразу или приватный ключ ни в GitHub, ни в Render, ни в `CASES_JSON`.

Приложению нужен только публичный адрес. Переводы сразу идут на этот адрес; отдельный бот-кастодиан не нужен.

## 4. Получите ключ TON Center

1. Откройте официальный сайт TON Center: `https://toncenter.com/`.
2. Создайте API key для Mainnet в соответствии с текущей инструкцией сервиса.
3. Скопируйте ключ и сохраните временно в менеджере паролей.
4. Не публикуйте его в GitHub. Без ключа код умеет работать с публичным лимитом, но для стабильного production-поллинга ключ обязателен.

## 5. Загрузите новые файлы в GitHub

1. Откройте репозиторий `denixl-11/dnx-store`.
2. Перейдите на ветку `main`.
3. Нажмите **Add file** → **Upload files**.
4. Перетащите в окно все эти файлы:
   - `index.html`
   - `main.py`
   - `requirements.txt`
   - `.python-version`
   - `.gitignore`
   - `README.md`
   - `DEPLOYMENT_RU.md`
   - `app-icon.png`
   - `tonconnect-manifest.json`
   - `terms.html`
   - `privacy.html`
5. Проверьте, что рядом с существующими файлами GitHub пишет **modified**, а новые файлы перечислены как добавленные.
6. В поле сообщения коммита введите `TON Connect and neon redesign`.
7. Выберите **Commit directly to the main branch**.
8. Нажмите **Commit changes**.
9. Никогда не загружайте `.env`, ZIP-архив, дамп базы или файл с ключами.

## 6. Проверьте GitHub Pages и TON manifest

1. В репозитории откройте **Settings** → **Pages**.
2. В **Build and deployment** выберите **Deploy from a branch**.
3. Выберите branch `main`, folder `/ (root)` и нажмите **Save**.
4. Откройте вкладку **Actions** и дождитесь зелёной галочки у Pages deployment.
5. По очереди откройте в браузере:
   - `https://denixl-11.github.io/dnx-store/`
   - `https://denixl-11.github.io/dnx-store/tonconnect-manifest.json`
   - `https://denixl-11.github.io/dnx-store/app-icon.png`
   - `https://denixl-11.github.io/dnx-store/terms.html`
   - `https://denixl-11.github.io/dnx-store/privacy.html`
6. Manifest должен показывать JSON, иконка — прежний знак приложения с неоновой рамкой, а обе страницы — текст. Если GitHub показывает 404, подождите 2–5 минут и обновите страницу.

## 7. Проверьте подключение Neon

1. В Neon Console откройте проект → **Dashboard** → **Connect**.
2. Выберите базу `neondb` и роль `neondb_owner`.
3. Включите **Connection pooling**. В hostname должно быть `-pooler`.
4. Скопируйте всю строку `postgresql://.../neondb?sslmode=require`.
5. Строку не публикуйте. Она понадобится только в Render Environment.

## 8. Один раз переведите старые рублёвые данные в TON

Старые числа в базе были рублями. Новый код воспринимает те же столбцы как TON, поэтому перед первым production-запуском выберите один фиксированный курс миграции. Пример: если вы решили считать `1 TON = 300 RUB`, коэффициент равен `300`.

1. В Neon откройте **SQL Editor**.
2. Замените каждое число `300` ниже на выбранный вами курс RUB за 1 TON.
3. Выполните запрос один раз:

```sql
BEGIN;
UPDATE users SET balance = ROUND(balance / 300, 9);
UPDATE items SET price = ROUND(price / 300, 9);
UPDATE cases SET price = ROUND(price / 300, 9);
UPDATE case_drops SET value = ROUND(value / 300, 9);
COMMIT;
```

4. Убедитесь, что запрос завершился без ошибки.
5. Не запускайте его повторно: повторный запуск ещё раз поделит значения.
6. Старую `game_history` лучше не пересчитывать: это архив уже завершённых раундов. Если хотите сохранить старую историю в рублях, очистите её до запуска: `TRUNCATE game_history;` — только если история вам не нужна.

### Отдельно измените CASES_JSON

`CASES_JSON` хранится в Render, а не в таблицах. Для каждого кейса вручную пересчитайте:

- `price`: старая цена в RUB / выбранный курс;
- у каждого drop поле `value`: старое значение в RUB / выбранный курс;
- `chance` и `real_chance` не меняйте;
- id, имена, картинки и модель не меняйте.

Пример при курсе 300: цена `150 RUB` становится `0.5`, значение `75 RUB` становится `0.25`.

## 9. Настройте Render Environment

1. Откройте Render Dashboard.
2. Выберите ваш Web Service `dnx-shop`.
3. В левом меню нажмите **Environment**.
4. Добавьте или обновите значения:

| Key | Что вставить |
|---|---|
| `BOT_TOKEN` | Текущий токен именно этого бота из BotFather |
| `ADMIN_ID` | Ваш числовой Telegram ID |
| `DATABASE_URL` | Pooled-строка Neon из шага 7 |
| `TON_RECEIVER_ADDRESS` | Публичный адрес кошелька администратора из шага 3 |
| `TONCENTER_API_KEY` | Ключ TON Center из шага 4 |
| `TONCENTER_API_URL` | `https://toncenter.com/api/v2` |
| `TON_MIN_DEPOSIT` | `0.05` |
| `TON_DEPOSIT_TIMEOUT` | `900` |
| `TON_MIN_BET` | `0.1` |
| `CASES_JSON` | Пересчитанный JSON из шага 8 |
| `WEBAPP_URL` | `https://denixl-11.github.io/dnx-store/` |
| `CORS_ORIGIN` | `https://denixl-11.github.io` без завершающего `/` |
| `DB_POOL_MIN_SIZE` | `1` |
| `DB_POOL_MAX_SIZE` | `10` |
| `INIT_DATA_MAX_AGE` | `86400` |
| `PYTHON_VERSION` | `3.14.3` |

5. Удалите старую переменную `PAYMENT_REQUISITES`: новый код её не использует.
6. `DB_PASSWORD` после успешного запуска можно удалить, потому что `DATABASE_URL` содержит все параметры подключения.
7. Ещё раз проверьте: в Environment не должно быть seed-фразы и приватного ключа.
8. Нажмите **Save, rebuild, and deploy**.

## 10. Настройте Render Service

1. Откройте сервис → **Settings**.
2. Build Command: `pip install -r requirements.txt`.
3. Start Command: `python main.py`.
4. Health Check Path: `/health`.
5. Оставьте **ровно один instance**. Игровой раунд и Telegram polling работают в одном процессе; два instance создадут два конкурирующих состояния.
6. Если Render использует старый кэш, откройте **Manual Deploy** → **Clear build cache & deploy**.

При первом запуске код автоматически создаст таблицу `ton_deposits` и нужные индексы. Существующие предметы и пользователи не удаляются.

## 11. Первый безопасный тест

1. В Render откройте **Logs**.
2. Найдите строку `DB initialized`. Не должно быть `DB Init Error`.
3. Откройте `https://dnx-shop.onrender.com/health`. Ответ должен быть `{"status":"ok"}`.
4. Откройте Mini App только через кнопку бота в Telegram. В обычном браузере защищённые запросы не работают, потому что там нет подписанного `initData`.
5. Нажмите `+` рядом с балансом.
6. Выберите `0.05 TON` или введите `0.05`.
7. Нажмите **Подключить кошелёк**, выберите официальный Telegram Wallet или другой поддерживаемый TON Connect кошелёк.
8. Нажмите **Пополнить** и подтвердите транзакцию в кошельке.
9. Не редактируйте точную сумму, которую показывает кошелёк: к выбранной сумме добавляется очень маленькая уникальная часть, по которой сервер безопасно различает платежи.
10. Вернитесь в Mini App. После подтверждения блокчейном баланс обновится автоматически, обычно в течение нескольких секунд.
11. В Render Logs должна появиться строка `Credited 1 TON deposit(s)`.
12. Проверьте повторно через минуту: баланс не должен зачислиться второй раз.

## 12. Полная функциональная проверка

Выполняйте дешёвыми тестовыми значениями:

1. Баланс отображается в TON и не показывает знак рубля.
2. Каталог: открыть карточку, добавить в корзину, купить один предмет.
3. Инвентарь: купленный предмет появился ровно один раз.
4. Кейсы: открыть дешёвый кейс, получить один drop, продать тестовый drop.
5. Игра: поставить `0.1 TON` с двух тестовых аккаунтов, дождаться победителя.
6. Убедиться, что формула вероятностей и поведение раунда не изменились.
7. Проверить золотую рамку победителя у каждой внешней стороны поля: свечение не должно прятаться под рамкой canvas.
8. Проверить вкладку истории и пустые состояния инвентаря.
9. Закрыть и заново открыть Mini App: TON-кошелёк должен восстановить сессию, незавершённый депозит — продолжить проверку.

## 13. Что делать при ошибке

- **`ton_deposits_disabled`** — в Render отсутствует `TON_RECEIVER_ADDRESS` или сервис не был redeploy после добавления.
- **`invalid_wallet`** — переподключите кошелёк в окне TON Connect и не вставляйте адрес вручную.
- **`ton_payment_error`** — проверьте `TONCENTER_API_URL`, API key и Render Logs.
- **Платёж ушёл, баланс не зачислен** — не создавайте второй платёж. Сохраните hash транзакции, адрес отправителя и сумму; проверьте, что перевод был в TON Mainnet и именно на `TON_RECEIVER_ADDRESS`.
- **`invalid_signature`** — полностью закройте Mini App, откройте заново из правильного бота и сверьте `BOT_TOKEN`.
- **`Database pool is not initialized` / `DB Init Error`** — проверьте pooled `DATABASE_URL`, пароль, ветку Neon и `sslmode=require`.
- **GitHub Pages показывает старый дизайн** — откройте **Actions**, дождитесь Pages deploy, затем перезапустите Mini App. Telegram WebView может кэшировать страницу несколько минут.
- **TON Connect не показывает кошельки** — сначала проверьте четыре публичных URL из шага 6 и точное совпадение домена в manifest.

## 14. Эксплуатация

1. Раз в день просматривайте Render Logs на `TON deposit scan failed`.
2. Не меняйте `TON_RECEIVER_ADDRESS`, пока существуют отправленные, но ещё не зачисленные платежи.
3. При ротации TON Center API key меняйте только Environment и делайте redeploy.
4. Перед любым изменением цен создавайте резервную копию Neon.
5. Никогда не подтверждайте баланс вручную по скриншоту пользователя: доказательство платежа — транзакция в блокчейне и запись `credited` в `ton_deposits`.
