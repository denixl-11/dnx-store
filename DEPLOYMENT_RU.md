# DNX Store: установка версии V5 Stars + TON

Эта инструкция рассчитана на работу без терминала. Выполняйте пункты строго по порядку.

## 1. Что находится в новой версии

1. Баланс пользователей, цены товаров, кейсов, стоимость ставок и выигрыши отображаются в Telegram Stars.
2. Все значения Stars должны быть целыми числами.
3. Пользователь может напрямую оплатить Stars через официальный Telegram invoice `XTR`.
4. После `successful_payment` сервер проверяет пользователя, payload, сумму и уникальный `telegram_payment_charge_id`, затем атомарно зачисляет Stars.
5. TON Connect остаётся альтернативным способом: `1 TON = 85 Stars`.
6. При TON-платеже сервер сам проверяет блокчейн; сообщение браузера не считается доказательством оплаты.
7. Вариант «Рубли» виден, но отключён и подписан «Скоро».
8. Аватары пользователей и изображения продуктов не изменяются.
9. Для оформления добавлены отдельные фоны игрового поля и истории.
10. Исправлена safe area на Android: приложение не добавляет искусственные 70–105 px и не форсирует fullscreen на Android.
11. Шапка, страницы, корзина и нижняя навигация ограничены единым контейнером до 480 px на больших экранах.
12. Каталог, кейсы и инвентарь показывают ровно две карточки в строке, включая узкие устройства.
13. Игровое поле квадратное и совпадает по ширине с информацией и кнопками.
14. В каталоге появился выключенный по умолчанию фильтр по полю `name` NFT: например, Chill Flame или Big Year.
15. После пополнения Mini App показывает отдельное окно успеха, а бот отправляет премиальное сообщение в чат.
16. Во всём интерфейсе используется эталонная Telegram-звезда из предоставленного WEBP. Она встроена в `index.html`, поэтому отдельный файл звезды загружать не требуется.
17. Верхний отступ учитывает встроенные кнопки Telegram в полноэкранном режиме.
18. Во время ожидания игроков космические линии и тонкие молнии плавно пульсируют без блика, пересекающего всё поле.
19. В инвентаре карточка показывает только `name`; `model` находится в характеристиках. Ring, Diamond и Cup не открывают окно характеристик.
20. Нижняя навигация примыкает к краю экрана; нижняя safe area учитывается внутри панели и больше не создаёт внешний зазор.
21. В модальном окне предмета Telegram-ссылка оставлена только в цветной кнопке сверху слева.
22. Сумма везде показывается в порядке `число → звезда`.
23. Игровая палитра содержит 32 контрастных варианта по четыре цвета; тёмный и светлый контуры не дают соседним полигонам сливаться.
24. Последние три секунды движения камера плавно приближает мяч до `1.65×`, следует за ним и возвращает полный обзор до золотой подсветки.

Telegram официально требует Stars для цифровых товаров. Прямой способ `XTR` соответствует этому требованию. TON остаётся дополнительным способом на вашей ответственности.

## 2. Сделайте резервную копию

1. Откройте `https://github.com/denixl-11/dnx-store`.
2. Нажмите **Code** → **Download ZIP**.
3. Сохраните архив отдельно и не перезаписывайте его.
4. В Neon Console убедитесь, что выбран production-проект.
5. В Render откройте **Environment** и сохраните копию текущего `CASES_JSON` в локальный текстовый файл.

## 3. Определите, в каких единицах сейчас лежат числа

Новый код не угадывает происхождение старых значений.

### Вариант A: предыдущая TON-версия ещё не разворачивалась

Если в базе цена товара `400` и вы хотите, чтобы после обновления она стала `400 Stars`, SQL-конвертация не нужна. Число останется тем же, изменится валюта интерфейса.

Проверьте, чтобы цены и баланс были целыми числами. В Neon SQL Editor один раз выполните:

```sql
BEGIN;
UPDATE users SET balance = ROUND(balance);
UPDATE items SET price = ROUND(price);
UPDATE cases SET price = ROUND(price);
UPDATE case_drops SET value = ROUND(value);
UPDATE game_history SET win_amount = ROUND(win_amount);
COMMIT;
```

### Вариант B: предыдущая TON-версия уже работала

Если значение `1` в базе сейчас означает `1 TON`, его нужно превратить в `85 Stars`. Выполните следующий запрос только один раз:

```sql
BEGIN;
UPDATE users SET balance = ROUND(balance * 85);
UPDATE items SET price = ROUND(price * 85);
UPDATE cases SET price = ROUND(price * 85);
UPDATE case_drops SET value = ROUND(value * 85);
UPDATE game_history SET win_amount = ROUND(win_amount * 85);
COMMIT;
```

Не запускайте оба варианта и не запускайте запрос повторно.

## 4. Обновите CASES_JSON

`CASES_JSON` хранится в Render и не изменяется SQL-запросом.

1. Откройте сохранённую копию JSON.
2. Для каждого кейса проверьте `price`.
3. Для каждого drop проверьте `value`.
4. Оба поля должны быть целыми Stars.
5. `chance`, `real_chance`, id, имена и ссылки на изображения не меняйте.
6. Если JSON уже был переведён на TON, умножьте `price` и `value` на `85` и округлите до целого.
7. Если старые числа должны просто стать Stars, оставьте их и удалите дробную часть.

Пример корректного фрагмента:

```json
{
  "1": {
    "id": 1,
    "name": "Спокойное Пламя",
    "price": 249,
    "image_url": "https://example.com/case.png",
    "drops": [
      {
        "id": 1,
        "name": "Chill Flame",
        "value": 120,
        "chance": 25,
        "real_chance": 25,
        "image_url": "https://example.com/item.png"
      }
    ]
  }
}
```

## 7. Подготовьте Neon

1. Neon Console → проект → **Connect**.
2. Выберите `neondb` и `neondb_owner`.
3. Включите **Connection pooling**.
4. Скопируйте строку, hostname которой содержит `-pooler`.
5. Не публикуйте строку.

При первом запуске код создаст `star_payments`, добавит `credit_stars` в `ton_deposits` и создаст индексы. Существующие пользователи и предметы не удаляются.

## 8. Настройте Render Environment

Render Dashboard → сервис `dnx-shop` → **Environment**.

Добавьте или обновите:

| Key | Value |
|---|---|
| `BOT_TOKEN` | Токен вашего бота от BotFather |
| `ADMIN_ID` | Числовой Telegram ID администратора |
| `DATABASE_URL` | Pooled-строка Neon |
| `CASES_JSON` | JSON с целыми ценами Stars |
| `WEBAPP_URL` | `https://denixl-11.github.io/dnx-store/` |
| `CORS_ORIGIN` | `https://denixl-11.github.io` |
| `TON_RECEIVER_ADDRESS` | Публичный адрес кошелька администратора |
| `TONCENTER_API_KEY` | Ключ TON Center Mainnet |
| `TONCENTER_API_URL` | `https://toncenter.com/api/v2` |
| `TON_STAR_RATE` | `85` |
| `TON_DEPOSIT_TIMEOUT` | `900` |
| `STAR_MIN_TOPUP` | `10` |
| `STAR_MAX_TOPUP` | `10000` |
| `STAR_MIN_BET` | `10` |
| `STAR_BET_STEP` | `10` |
| `DB_POOL_MIN_SIZE` | `1` |
| `DB_POOL_MAX_SIZE` | `10` |
| `INIT_DATA_MAX_AGE` | `86400` |
| `PYTHON_VERSION` | `3.14.3` |

Удалите устаревшие `TON_MIN_DEPOSIT`, `TON_MIN_BET` и `PAYMENT_REQUISITES`.

Для Telegram Stars не нужен provider token. Не создавайте сторонний платёжный токен и не добавляйте его в Environment.

Нажмите **Save, rebuild, and deploy**.

## 10. Тест прямой оплаты Telegram Stars

1. Откройте Mini App из кнопки бота.
2. Нажмите `+` у баланса.
3. Выберите, например, `85 Stars`.
4. Нажмите **Telegram Stars**.
5. Telegram откроет официальный счёт `XTR`.
6. Подтвердите оплату.
7. Дождитесь сообщения бота о зачислении.
8. Вернитесь в приложение: баланс должен увеличиться ровно на `85`.
9. Повторное открытие того же счёта не должно дать повторное зачисление.
10. Команда `/paysupport` должна вернуть инструкцию для обращения по платежу.

## 11. Тест TON

1. Выберите `85 Stars`.
2. Нажмите **Toncoin**.
3. Интерфейс покажет примерно `1 TON`.
4. Подключите TON Connect кошелёк.
5. Подтвердите точную сумму из кошелька. Она может быть немного больше расчётной из-за уникального платёжного маркера.
6. После подтверждения блокчейна должно зачислиться ровно `85 Stars`, а не сумма TON.
7. Проверьте Render Logs: должна появиться строка `Credited 1 TON deposit(s)`.

## 12. Проверьте дизайн и функции

1. Игровое поле квадратное, с короной, пульсирующими космическими линиями, тонкими молниями и текстом ожидания.
2. Аватары игроков внутри игры не изменены.
3. Формулы игры и определение победителя не изменены.
4. Каталог использует исходные изображения товаров.
5. Активный таб каталога — purple/magenta, кейсов — red/magenta.
6. Инвентарь использует исходные изображения предметов.
7. Пустая история показывает большие фиолетовые песочные часы.
8. Все цены, кейсы, ставки, выигрыши и продажи отображаются в Stars.
9. В способах пополнения есть Telegram Stars, TON и отключённые рубли «Скоро».
10. Проверьте покупку, корзину, кейс, продажу drop, вывод NFT и игру с двух аккаунтов.
11. На Android сверху не должно быть большого чёрного блока, а все кнопки должны оставаться видимыми и нажимаемыми.
12. На ноутбуке шапка, страница и нижняя панель должны иметь одинаковую ширину; корзина должна находиться внутри правого края страницы.
13. В каталоге выберите NFT по названию (например, Chill Flame), проверьте результат, затем нажмите `×` и убедитесь, что вернулись все товары.
14. После тестового пополнения проверьте одновременно окно внутри Mini App и сообщение от бота в чате.
15. Убедитесь, что в карточках инвентаря виден только `name`, а `model` появляется только в характеристиках.
16. Нажмите Ring, Diamond и Cup: окно характеристик открываться не должно; кнопка «Вывести» должна работать как раньше.
17. Проверьте нижнюю панель: под ней не должно быть полосы фона, а иконки и подписи не должны обрезаться.
18. Откройте предмет со ссылкой `nft_link`: кнопка Telegram должна быть только сверху слева, рядом с названием второй ссылки быть не должно.
19. Проверьте, что во всех суммах сначала стоит число, затем сине-фиолетово-розовая звезда.
20. В игре с несколькими участниками каждый полигон должен иметь четырёхцветный градиент и отчётливые разделительные линии.
21. В последние три секунды движения поле должно плавно приблизиться к мячу до `1.65×`, а перед золотой подсветкой полностью вернуться к общему виду.
22. Нижняя розовая линия должна быть видна у панели навигации и у окна характеристик, при этом под панелью не должно быть внешнего зазора.

## 13. Возможные ошибки

- `invalid_signature` — закройте Mini App и откройте заново из правильного бота; проверьте `BOT_TOKEN`.
- `stars_invoice_error` — проверьте Bot token и Render Logs; provider token для XTR не нужен.
- Счёт оплачен, но баланс не обновился — проверьте, пришёл ли боту `successful_payment`, и найдите запись в `star_payments`.
- `ton_deposits_disabled` — не задан `TON_RECEIVER_ADDRESS`.
- `invalid_wallet` — переподключите кошелёк TON Connect.
- `DB Init Error` — проверьте pooled `DATABASE_URL`.
- Картинки не видны — проверьте Pages deployment и три PNG-адреса из шага 6.
- Старые цены выглядят слишком маленькими — вероятно, TON-значения не были умножены на `85`.
- Старые цены стали слишком большими — вероятно, миграция была выполнена дважды.

## 14. Безопасность

1. Никогда не храните seed-фразу или приватный ключ в Render.
2. Не зачисляйте Stars по скриншоту.
3. Для Stars доказательством является `successful_payment` и уникальный `telegram_payment_charge_id`.
4. Для TON доказательством является транзакция блокчейна, найденная сервером.
5. Не меняйте `TON_RECEIVER_ADDRESS`, пока есть ожидающие TON-платежи.
6. Регулярно просматривайте Render Logs и делайте резервные копии Neon.
