DNX STORE V8 — ПОЛНАЯ ИНСТРУКЦИЯ ПО ОБНОВЛЕНИЮ
================================================

ЧТО ИЗМЕНИЛОСЬ В V8
--------------------
1. Основной баланс отображается жёлтой звездой, бонусный — оригинальной
   фиолетово-синей звездой из версии 6.5.
2. На странице баланса основной блок занимает 75%, бонусный — 25%.
3. Каталог получил поиск по названию: можно печатать с клавиатуры и выбирать
   только совпадающие NFT.
4. Покупка из каталога и корзины стала идемпотентной: повторный ответ сети не
   может повторно списать баланс. Ошибка обновления интерфейса больше не
   показывается как ошибка уже завершённой покупки.
5. Предметы из кейсов можно продать. Они не удаляются из базы, а переводятся в
   статус disposed и остаются в журнале событий.
6. Купленные в каталоге NFT нельзя перепродать. Кнопка показывает объяснение.
7. После запроса вывода действует двухчасовой таймер. Кнопки вывода и продажи
   блокируются до окончания срока.
8. История игр, кейсы и страница балансов получили премиальное оформление.
9. Камера игры приближает мяч последние 4 секунды до 1.9x, плавно возвращается,
   выдерживает 0.5 секунды общего плана и только затем показывает победителя.
10. Положение мяча и победитель не пересчитываются: изменилась только отрисовка.
11. В центре поля пульсирует только существующая обводка шестиугольника.
12. Добавлены служебные таблицы покупок и истории предметов.


ПЕРЕД ОБНОВЛЕНИЕМ — РЕЗЕРВНАЯ КОПИЯ
-----------------------------------
1. Откройте https://console.neon.tech/
2. Выберите production-проект DNX Store.
3. Откройте Branches.
4. Нажмите Create branch.
5. Назовите ветку backup-before-v8 и создайте её от текущей production-ветки.
6. Откройте https://dashboard.render.com/
7. Выберите Web Service магазина -> Environment.
8. Скопируйте CASES_JSON в отдельный текстовый файл.
9. Не удаляйте и не меняйте BOT_TOKEN, DATABASE_URL и остальные секреты.


КАК ЗАГРУЗИТЬ V8 В GITHUB БЕЗ ТЕРМИНАЛА
---------------------------------------
1. Распакуйте dnx-store-stars-final-v8.zip.
2. Откройте https://github.com/denixl-11/dnx-store
3. Переключитесь на ветку main.
4. Нажмите Add file -> Upload files.
5. Откройте распакованную папку dnx-store-stars-final-v8.
6. Выделите ВСЕ файлы внутри папки, включая:
   - index.html
   - main.py
   - requirements.txt
   - .python-version
   - .gitignore
   - main-star.png
   - bonus-star.webp
   - все PNG, JS, WASM, HTML и TXT-файлы.
7. Перетащите файлы в окно GitHub. Перетаскивайте содержимое папки, а не саму
   папку, иначе приложение окажется во вложенном каталоге и не запустится.
8. Дождитесь полной загрузки файлов.
9. В Commit message напишите: DNX Store V8.
10. Выберите Commit directly to main.
11. Нажмите Commit changes.
12. Не загружайте .env, пароли, токены, seed-фразы и резервные ZIP.


КАК ОБНОВИТЬ RENDER
-------------------
1. Откройте https://dashboard.render.com/
2. Выберите Web Service магазина.
3. Откройте Settings и проверьте:
   - Build Command: pip install -r requirements.txt
   - Start Command: python main.py
   - Health Check Path: /health
4. Если Auto-Deploy включён, откройте Events и дождитесь нового deploy.
5. Если Auto-Deploy выключен, нажмите Manual Deploy -> Deploy latest commit.
6. Для гарантированного обновления старого кэша можно один раз выбрать
   Manual Deploy -> Clear build cache & deploy.
7. Откройте Logs.
8. Дождитесь запуска HTTP-сервера и успешного init_db.
9. Откройте адрес /health. Ответ должен быть {"status":"ok"}.


ENVIRONMENT И NEON
------------------
Новых обязательных переменных Environment в V8 нет. Сохраните существующие:
- BOT_TOKEN
- ADMIN_ID
- DATABASE_URL (желательно pooled URL Neon с -pooler в hostname)
- WEBAPP_URL
- CORS_ORIGIN
- CASES_JSON
- TON_RECEIVER_ADDRESS
- TONCENTER_API_KEY
- TONCENTER_API_URL
- TON_STAR_RATE=85
- остальные текущие лимиты Stars, TON и DB pool.

Вручную запускать SQL не требуется. При первом запуске V8 автоматически:
- добавит items.acquisition_source;
- добавит withdraw_requested_at, withdraw_expires_at и disposed_at;
- создаст purchase_records;
- создаст item_events;
- создаст нужные индексы;
- пометит старые case_drop как источник case.

Существующие пользователи, балансы, NFT, кейсы и история не удаляются.


КАК ПРОВЕРИТЬ ПОКУПКУ
---------------------
1. Откройте Mini App из Telegram, а не прямой ссылкой браузера.
2. Откройте Магазин -> Каталог.
3. В поиске напечатайте часть имени, например chill.
4. Убедитесь, что неподходящие карточки скрылись.
5. Купите дешёвый тестовый NFT.
6. Основной баланс должен уменьшиться один раз.
7. NFT должен исчезнуть из каталога и появиться в Инвентаре.
8. Повторите проверку через корзину.
9. Если Render долго отвечает после сна, не нажимайте кнопку многократно:
   запрос имеет уникальный requestId и безопасно восстановит результат.
10. При настоящей серверной ошибке окно покажет код операции. Найдите тот же
    код в Render Logs рядом с Buy transaction failed.


КАК ПРОВЕРИТЬ ИНВЕНТАРЬ
-----------------------
1. Для NFT, купленного в каталоге, должны быть две кнопки:
   - ВЫВЕСТИ;
   - НЕЛЬЗЯ ПРОДАТЬ.
2. Нажмите НЕЛЬЗЯ ПРОДАТЬ. Должно появиться объяснение и кнопка ПРОДОЛЖИТЬ.
3. Для NFT, выпавшего из кейса, должны быть ВЫВЕСТИ и ПРОДАТЬ.
4. После продажи предмет должен исчезнуть из активных и появиться в истории со
   статусом ПРОДАНО. В Neon запись items остаётся со статусом disposed.
5. Нажмите ВЫВЕСТИ на тестовом NFT. Кнопка станет серой и покажет таймер 2:00:00.
6. До окончания таймера повторный вывод и продажа заблокированы.
7. Администратор должен подтвердить вывод в сообщении бота в течение 2 часов.
8. Если время вышло, пользователь сможет отправить новый запрос.


КАК ПРОВЕРИТЬ ИГРУ
------------------
1. До запуска раунда в центре должна пульсировать только рамка существующего
   шестиугольника с короной. Дополнительных шестиугольников быть не должно.
2. Проведите тестовый раунд минимум с двумя игроками.
3. Первые 6 секунд движения виден общий план.
4. Последние 4 секунды камера очень плавно приближает мяч до 1.9x и следует за ним.
5. После остановки камера плавно возвращает весь квадрат поля.
6. Около 0.5 секунды остаётся общий план без золотой рамки.
7. Затем появляется золотая обводка победителя.
8. Конечная координата и победитель должны совпадать с серверными данными.


ФИНАЛЬНАЯ ПРОВЕРКА УСТРОЙСТВ
----------------------------
1. iPhone: каталог, баланс, игра, инвентарь, история, кейс, обе NFT-модалки.
2. Android: те же экраны; не должно быть чёрной половины или невидимых кнопок.
3. Mac/Windows: контейнер должен быть по центру, поле квадратное, содержимое кейса
   не растянуто.
4. На всех устройствах нижняя панель должна быть замкнутым скруглённым
   прямоугольником без бокового обрезания и внешнего зазора.
5. Убедитесь, что index.html и main.py загружены из одного архива V8.


КАК ОТКАТИТЬСЯ
--------------
1. GitHub -> Commits -> откройте коммит перед DNX Store V8.
2. Нажмите Revert или загрузите сохранённый резервный ZIP.
3. Render -> Manual Deploy -> Deploy latest commit.
4. Новые поля Neon можно оставить: старому коду они не мешают.
