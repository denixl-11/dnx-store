DNX STORE V7 — ИНСТРУКЦИЯ ПО УСТАНОВКЕ
========================================

ВАЖНО ПЕРЕД ОБНОВЛЕНИЕМ
-----------------------
1. Откройте Neon: https://console.neon.tech/
2. Выберите проект вашего магазина.
3. Откройте раздел Branches и создайте резервную ветку базы данных.
   Назовите её, например: backup-before-v7.
4. Не удаляйте и не изменяйте существующие переменные Environment в Render.


КАК ЗАГРУЗИТЬ V7 В GITHUB
-------------------------
1. Распакуйте архив dnx-store-stars-final-v7.zip.
2. Откройте репозиторий https://github.com/denixl-11/dnx-store
3. Нажмите Add file.
4. Нажмите Upload files.
5. Перетащите в окно ВСЁ содержимое папки dnx-store-stars-final-v7.
   Файлы должны заменить старые файлы в корне репозитория.
6. Внизу в поле Commit changes напишите: DNX Store V7.
7. Нажмите Commit changes.


КАК ОБНОВИТЬ RENDER
-------------------
1. Откройте https://dashboard.render.com/
2. Выберите Web Service вашего магазина.
3. Если Auto-Deploy включён, дождитесь окончания нового Deploy.
4. Если Auto-Deploy выключен:
   - нажмите Manual Deploy;
   - выберите Deploy latest commit.
5. Откройте вкладку Logs.
6. Дождитесь строк о запуске приложения и успешной инициализации БД.
7. Откройте вкладку Environment и убедитесь, что старые значения сохранены.


ПЕРЕМЕННЫЕ ENVIRONMENT
----------------------
Обязательных новых секретов для V7 нет.

Не удаляйте существующие:
- BOT_TOKEN
- DATABASE_URL или параметры подключения Neon
- WEBAPP_URL
- ADMIN_ID
- TON_RECEIVER_ADDRESS и другие TON-параметры
- CASES_JSON

Необязательная переменная:
- BOT_USERNAME — имя бота без символа @, например DnxStoreBot.

Если BOT_USERNAME не задан, приложение автоматически запросит имя через
официальный Telegram Bot API при запуске. Переменная нужна только как запасной
вариант, если на Render временно нет доступа к Telegram.


АВТОМАТИЧЕСКИЕ ИЗМЕНЕНИЯ NEON
-----------------------------
При первом запуске V7 само создаст недостающие поля и индексы:
- users.bonus_balance
- users.referral_code
- users.referred_by
- таблицу referral_rewards
- недостающие служебные поля items, включая items.last_event

Вручную выполнять SQL или переносить данные не требуется.
Старые пользователи и предметы сохраняются.


РЕФЕРАЛЬНАЯ СИСТЕМА
-------------------
1. Каждый пользователь получает уникальный код из 25 символов.
2. В разделе пополнения отображается его персональная Telegram-ссылка.
3. Пользователь нажимает «СКОПИРОВАТЬ ССЫЛКУ» и отправляет её другу.
4. Реферал должен быть НОВЫМ пользователем и впервые запустить бота именно по
   этой ссылке.
5. Один пользователь может быть закреплён только за одним пригласившим.
6. При каждом успешном пополнении реферала пригласившему один раз начисляется
   5% на бонусный баланс.
7. Повторная обработка одного и того же платежа не создаёт повторный бонус.


ОСНОВНОЙ И БОНУСНЫЙ БАЛАНС
--------------------------
- Основной баланс отмечен жёлтой звездой.
- Бонусный баланс отмечен розово-фиолетовой звездой.
- NFT из каталога можно покупать только за основной баланс.
- В игре баланс выбирается перед подтверждением ставки.
- При ставке с основного баланса победитель получает свою ставку и 70% чужих.
- При ставке с бонусного баланса победитель получает на ОСНОВНОЙ баланс свою
  ставку и 40% чужих ставок.
- Отменённая одиночная ставка возвращается на тот баланс, с которого сделана.
- Продажа предмета из кейса всегда начисляет средства на основной баланс.
- Бонусный баланс нельзя пополнить вручную — только реферальными начислениями.


КЕЙСЫ И ОПЛАТА БОНУСАМИ
-----------------------
По умолчанию все существующие кейсы оплачиваются только основным балансом.
Чтобы разрешить конкретный кейс за бонусы, добавьте в объект этого кейса в
CASES_JSON поле:

"bonus_enabled": true

Пример:
{
  "id": 1,
  "name": "Chill Flame Case",
  "price": 249,
  "bonus_enabled": true,
  "image_url": "https://...",
  "drops": [ ... ]
}

Для кейса без поля bonus_enabled или со значением false переключатель оплаты
бонусами не показывается.


ПРОВЕРКА ПОСЛЕ DEPLOY
---------------------
1. Откройте бота на iPhone, Android и компьютере.
2. Проверьте, что основной баланс показан жёлтой звездой, бонусный — фиолетовой.
3. Откройте каталог и купите самый дешёвый тестовый NFT:
   - сумма должна списаться с основного баланса;
   - NFT должен исчезнуть из каталога;
   - NFT должен появиться в инвентаре.
4. Добавьте два предмета в корзину и повторите покупку.
5. Откройте таблицу лидеров: должны отображаться только места 1, 2 и 3.
6. Создайте реферальную ссылку на первом аккаунте.
7. Откройте её НОВЫМ тестовым Telegram-аккаунтом и запустите бота.
8. Пополните новый аккаунт на небольшую сумму.
9. На первом аккаунте должно появиться 5% на бонусном балансе.
10. Сделайте тестовую ставку бонусами. При победе выигрыш должен прийти на
    основной баланс по формуле: своя ставка + 40% чужих ставок.
11. Проверьте основной и разрешённый бонусный способ оплаты кейса.


ЕСЛИ ПОКУПКА СНОВА ПОКАЗЫВАЕТ ОШИБКУ
------------------------------------
1. Не нажимайте покупку много раз подряд.
2. Откройте Render -> ваш Web Service -> Logs.
3. Найдите строку «Buy transaction failed».
4. Убедитесь, что Deploy V7 завершился после загрузки файлов.
5. Проверьте, что в логах была успешная инициализация БД.
6. Если ошибка осталась, сохраните строку ошибки из Render Logs — теперь она
   содержит точную причину, а приложение не скрывает её за сообщением «сеть».
