# Промежуточная версия V8.9 — animation hotfix

Дата сборки: 23 июля 2026 года.

## Что завершено в этой сборке

- Убрата фоновая преждевременная подготовка скрытых NFT из страницы кейса, которая конкурировала за декодеры с видимыми NFT.
- Видимые NFT получили приоритет: подготовка постеров кейса начинается только после завершения стартовой волны видимых анимаций либо по короткому защитному таймауту.
- Состояние рулетки при первом открытии кейса сначала создаётся без скрытой тяжёлой подготовки, затем безопасно обновляется.
- Сокращено время ожидания синхронного старта группы NFT, чтобы один медленный ресурс не удерживал остальные карточки слишком долго.
- Технические заглушки карточек оставлены прозрачными до готовности медиа, чтобы не появлялись вложенные серые или фиолетовые квадраты.
- Добавлен безопасный запуск интерфейса вне Telegram WebApp для локальной браузерной проверки.
- Игровая механика и её расчёты в этом hotfix не изменялись.

## Выполненные проверки

- Python: `main.py` успешно проходит `py_compile`.
- JavaScript: встроенный скрипт `index.html` успешно проходит синтаксический разбор.
- Diff: ошибок пробелов и конфликтных маркеров не найдено.
- Локальный браузер: документ полностью загружается без фатального JavaScript-сбоя. Ошибки HTTP 404 при статическом запуске ожидаемы, поскольку API и база в таком режиме не запущены.

## Что обязательно проверить и при необходимости доделать следующим проходом

1. Реальные Telegram WebView: iPhone, Android и macOS/Windows Desktop.
2. Несколько циклов `Каталог → Кейсы → Каталог` и `Профиль → Магазин → Профиль` без исчезновения анимаций.
3. Одновременный старт всех видимых NFT и один полный цикл анимации после каждого повторного появления в viewport.
4. Повтор анимации в модальном окне по тапу и наведению мыши.
5. Содержимое кейса: единый фон `#242424`, модель примерно 70%, отсутствие вложенного квадрата и корректная работа на слабой сети.
6. Рулетка кейса: стабильный нулевой кадр без живых TGS-декодеров во время вращения и без увеличения модели до 95%.
7. Победивший предмет после рулетки: полноценная анимация и корректное повторное воспроизведение.
8. Профилирование памяти, количества canvas/WebGL-контекстов и частоты кадров на слабом Android.
9. Полная интеграционная проверка на Render с настоящими API, Neon и Telegram initData.

Эта сборка является аккуратно зафиксированным промежуточным hotfix, а не заявлением о завершённом тестировании на всех физических устройствах.
