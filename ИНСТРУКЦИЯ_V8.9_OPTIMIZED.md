# DNX Store V8.9 Optimized — установка и проверка

Это оптимизированная сборка на основе V8.9. Игровые формулы, определение победителя, цены, шансы кейсов и существующий пользовательский сценарий не изменялись.

## Что улучшено

- запросы к Neon выполняются через ограниченный асинхронный пул соединений;
- критические покупки, открытие и продажа кейсов, ставки, выплаты и выводы защищены транзакциями и повторными запросами;
- все видимые NFT-анимации запускаются сразу без искусственного лимита; скрытые плееры ставятся на паузу и не расходуют ресурсы;
- каталог, инвентарь и кейсы не запускают лишние одинаковые запросы;
- игровое поле кеширует статическую геометрию и перерисовывает её только при изменении участников;
- тяжёлые декоративные эффекты отключаются на слабых устройствах, но NFT, рулетка кейса и игровая анимация сохраняются;
- проценты выпадения уменьшены и закреплены в правом верхнем углу изображения предмета;
- кнопки и нижняя навигация проверены на ширинах от 320 до 1440 px;
- сервер проверяет Telegram `initData`, Origin, размеры запросов и обязательные переменные окружения;
- зависшие игровые ставки могут быть безопасно восстановлены после перезапуска Render.

## 1. Сделайте резервную копию

1. Откройте GitHub и перейдите в репозиторий `denixl-11/dnx-store`.
2. Нажмите зелёную кнопку **Code**.
3. Нажмите **Download ZIP** и сохраните архив текущей рабочей версии.
4. Не удаляйте старый архив, пока полностью не проверите новую сборку.

## 2. Загрузите новые файлы в GitHub

1. Распакуйте `dnx-store-stars-final-v8.9-opt.4-fixed.zip`.
2. Откройте распакованную папку `dnx-store-stars-final-v8.9-optimized`.
3. В GitHub откройте корень репозитория.
4. Нажмите **Add file → Upload files**.
5. Перетащите содержимое распакованной папки, а не саму внешнюю папку.
6. Убедитесь, что в корне GitHub находятся `index.html`, `main.py`, `requirements.txt`, `tgsticker.js`, `tgsticker-worker.js`, файлы WASM и изображения.
7. В поле сообщения коммита напишите `DNX Store V8.9 optimized`.
8. Нажмите **Commit changes**.

Если GitHub не заменяет старые файлы через браузер, удобнее открыть GitHub Desktop, скопировать содержимое папки в локальный репозиторий, проверить список изменений и нажать **Commit to main → Push origin**.

## 3. Проверьте Environment в Render

1. Откройте [Render Dashboard](https://dashboard.render.com/).
2. Выберите Web Service вашего бота.
3. В левом меню нажмите **Environment**.
4. Проверьте переменные:

| Переменная | Что указать |
|---|---|
| `BOT_TOKEN` | токен бота из BotFather |
| `ADMIN_ID` | ваш числовой Telegram ID |
| `DATABASE_URL` | строка подключения Neon с `postgresql://` |
| `WEBAPP_URL` | полный HTTPS-адрес Mini App, например `https://denixl-11.github.io/dnx-store/` |
| `CORS_ORIGIN` | только origin без пути, например `https://denixl-11.github.io` |
| `BOT_USERNAME` | username бота без `@` |
| `TON_RECEIVER_ADDRESS` | TON-адрес, принимающий пополнения |
| `TONCENTER_API_KEY` | ключ Toncenter, если он используется |
| `CASES_JSON` | существующая конфигурация кейсов |

Рекомендуемые дополнительные значения:

- `DB_POOL_MIN_SIZE=1`
- `DB_POOL_MAX_SIZE=6`
- `INIT_DATA_MAX_AGE=86400`
- `TON_DEPOSIT_TIMEOUT=900`
- `TON_STAR_RATE=85`
- `STAR_MIN_TOPUP=10`
- `STAR_MAX_TOPUP=10000`
- `STAR_MIN_BET=10`
- `STAR_BET_STEP=10`

Не добавляйте токены, пароли, ключи или `DATABASE_URL` в GitHub. Они должны оставаться только в Render Environment.

## 4. Настройте и перезапустите Render

1. В Render откройте **Settings**.
2. В поле **Build Command** укажите `pip install -r requirements.txt`.
3. В поле **Start Command** укажите `python main.py`.
4. В поле **Health Check Path** укажите `/health`.
5. Оставьте ровно **один экземпляр** сервиса. Текущее состояние игрового раунда хранится в памяти одного процесса, поэтому несколько экземпляров запускать нельзя.
6. Нажмите **Save Changes**.
7. Откройте **Manual Deploy** и нажмите **Deploy latest commit**.
8. Откройте вкладку **Logs** и дождитесь сообщения об успешном запуске HTTP-сервера и бота.

При первом запуске сервер сам создаст недостающие служебные таблицы и индексы. Вручную выполнять SQL-миграцию не требуется.

## 5. Проверьте GitHub Pages и BotFather

1. В GitHub откройте **Settings → Pages**.
2. Проверьте, что источник публикации — нужная ветка и папка `/root`.
3. Дождитесь зелёного статуса публикации.
4. Откройте адрес из `WEBAPP_URL` в браузере и убедитесь, что загружается свежий интерфейс.
5. В BotFather откройте `/mybots → ваш бот → Bot Settings → Menu Button`.
6. Проверьте, что кнопка ведёт на тот же `WEBAPP_URL`.

## 6. Обязательная проверка после обновления

1. Откройте `https://АДРЕС-RENDER/health`. Ответ должен содержать `"status": "ok"` и release `8.9-opt.4`.
2. Откройте Mini App на iPhone, Android и компьютере.
3. Переключите **Игра → Магазин → Профиль** несколько раз: не должно быть чёрного экрана, зависших анимаций и горизонтальной прокрутки.
4. В каталоге, корзине, инвентаре и кейсах проверьте все NFT в видимой области: они должны начинать анимацию сразу и одновременно, без очереди по 1–3 карточки.
5. Откройте корзину и удалите/добавьте товар.
6. Проведите одну тестовую покупку дешёвого товара: баланс должен списаться один раз, товар — один раз появиться в инвентаре.
7. Откройте кейс. Проценты должны находиться в правом верхнем углу изображения; рулетка и результат должны отображаться без обрезания.
8. Проверьте продажу приза из кейса и запрет продажи товара из каталога.
9. Проверьте запрос вывода и двухчасовой таймер блокировки повторного запроса.
10. Проведите тестовую игру двумя аккаунтами. Первая и повторная ставки должны приниматься; каждая ставка списывается ровно один раз, а результат и победитель совпадают с прежней логикой.
11. Сделайте маленькое тестовое пополнение Telegram Stars и TON.

## Если что-то пошло не так

1. Скопируйте последние строки из Render **Logs** до повторного запуска.
2. Проверьте `/health`.
3. Проверьте точное совпадение `WEBAPP_URL` и `CORS_ORIGIN` с адресом GitHub Pages.
4. Убедитесь, что Render запущен в одном экземпляре.
5. Для отката загрузите сохранённую V8.9 обратно в GitHub и нажмите **Deploy latest commit**. Новые служебные таблицы можно оставить: старой версии они не мешают.

## Автоматическая проверка Render каждые 5 минут

В сборку добавлен GitHub Actions workflow `.github/workflows/keep-render-awake.yml`. Он обращается только к публичному `/health`, поэтому Telegram-аккаунт, пароль, код подтверждения и session-файл не нужны.

1. Откройте файл `.github/workflows/keep-render-awake.yml` в GitHub.
2. В строке `RENDER_HEALTH_URL` уже указан `https://dnx-shop.onrender.com/health`. Если адрес вашего Render-сервиса другой, замените только hostname, сохранив `/health` в конце.
3. Сохраните файл через **Commit changes**.
4. Откройте в репозитории вкладку **Actions**.
5. Слева выберите **Keep Render health endpoint warm**.
6. Нажмите **Run workflow → Run workflow** для первой ручной проверки.
7. Откройте выполненный запуск: шаг `Ping public health endpoint` должен быть зелёным.

Workflow не проходит Telegram-авторизацию и не создаёт пользователя, поэтому строгая проверка `initData` его не блокирует. Никому не передавайте пароль Telegram, SMS-код, QR-сессию или файл сессии аккаунта.

## Важно

Локальная проверка подтверждает синтаксис Python и JavaScript, отсутствие горизонтального переполнения и корректную структуру интерфейса. Полную проверку реальных платежей, Neon и Telegram WebApp можно выполнить только после развёртывания с вашими Environment-переменными.
