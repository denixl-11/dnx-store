# DNX Store V8.9 opt.5 — исправление ставок и кейсов

Эта сборка должна загружаться на GitHub и Render целиком. Нельзя заменять только `index.html` или только `main.py`: клиент и сервер проверяют общий номер релиза `8.9-opt.5`.

## Что исправлено

- Ставка больше не ломается после перезапуска Render или при старом маркере миграции Neon.
- При запуске сервер сам проверяет и при необходимости создаёт служебные таблицы игрового раунда. Ручной SQL и изменение Environment не требуются.
- Незавершённые ставки старого процесса безопасно возвращаются, а занятый номер пустого раунда автоматически пропускается.
- Формулы выигрыша, игровая траектория, выбор победителя и остальная механика игры не изменены.
- В движущейся ленте кейса используются лёгкие нулевые кадры Telegram NFT. Поэтому рулетка не запускает десятки RLottie-анимаций одновременно и не показывает серые ячейки.
- После выпадения предмета снова показывается полноценная NFT-анимация. Анимации в каталоге, инвентаре, корзине и списке возможных призов сохранены.
- Нулевые кадры кейса заранее загружаются и декодируются при открытии страницы кейса.

## Как установить

1. Откройте репозиторий `denixl-11/dnx-store` на GitHub.
2. Нажмите **Add file → Upload files**.
3. Откройте папку этой сборки и загрузите **все файлы из неё**. Особенно важно заменить одновременно:
   - `main.py`
   - `index.html`
   - `tgsticker.js`
   - `tgsticker-worker.js`
4. Внизу страницы GitHub нажмите **Commit changes**.
5. Откройте Render → ваш Web Service → **Manual Deploy**.
6. Выберите **Clear build cache & deploy**, чтобы Telegram WebView и Render не получили смесь старого и нового JavaScript.
7. Дождитесь статуса **Live**.

Environment и структуру Neon вручную менять не нужно. Не удаляйте таблицы и данные пользователей.

## Обязательная проверка после публикации

1. Полностью закройте мини-приложение Telegram и откройте заново.
2. В игре поставьте 10 звёзд. Ставка должна появиться на поле без сообщения «Не удалось принять ставку».
3. Откройте кейс. В покоящейся и вращающейся ленте должны быть изображения нулевого кадра, а не серые квадраты.
4. После остановки победивший предмет должен открыться с полноценной анимацией.
5. Откройте каталог, корзину и профиль: все видимые NFT с `nft_link` должны анимироваться.

Если после деплоя Telegram показывает старую страницу, закройте мини-приложение, удалите его из недавних и откройте снова. На сервере и клиенте должен быть один релиз `8.9-opt.5`.
