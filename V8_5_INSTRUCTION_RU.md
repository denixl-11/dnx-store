# DNX Store V8.5 — инструкция

Этот файл специально имеет латинское имя, чтобы без проблем открываться на GitHub, Windows и macOS.

## 1. Сначала сделайте резервную копию Neon

1. Войдите в Neon и откройте проект базы данных.
2. Откройте **Branches**.
3. Нажмите **Create branch**.
4. Назовите ветку `before-v8-5` и дождитесь её создания.

При первом запуске V8.5 приложение автоматически:

- округлит основной и бонусный балансы вниз до целых звёзд;
- добавит в `items` столбцы `model`, `pattern`, `background`;
- перенесёт в них значения из старого `traits`;
- удалит старый столбец `items.traits` после успешного переноса.

## 2. Загрузите файлы в GitHub

1. Откройте репозиторий `denixl-11/dnx-store`.
2. Нажмите **Add file → Upload files**.
3. Перетащите все файлы из папки `dnx-store-stars-final-v8.5`.
4. Подтвердите замену файлов.
5. В поле сообщения напишите `DNX Store V8.5`.
6. Нажмите **Commit changes**.

Минимально обязательные файлы: `main.py`, `index.html`, `tgsticker.js`, `tgsticker-worker.js`, `rlottie-wasm.js`, `rlottie-wasm.wasm`, `pako-inflate.min.js` и все изображения.

## 3. Разверните на Render

1. Откройте Web Service бота в Render.
2. В **Settings** проверьте:
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python main.py`
   - Health Check Path: `/health`
3. Новые переменные Environment не нужны; существующие не удаляйте.
4. Нажмите **Manual Deploy → Deploy latest commit**.
5. В **Logs** дождитесь успешного запуска. Первый старт может быть дольше обычного из-за миграции базы.

## 4. Новые поля каталога

В таблице `items` используйте:

- `model` — модель;
- `pattern` — узор;
- `background` — фон.

Если значения нет, оставьте `NULL`.

## 5. Анимация NFT в кейсах

В `CASES_JSON` у нужного элемента `drops` добавьте:

```json
{
  "name": "Chill Flame",
  "image_url": "https://example.com/fallback.png",
  "nft_link": "https://t.me/nft/ChillFlame-12345",
  "model": "Bear Market",
  "pattern": "Oculus",
  "background": "Chocolate",
  "value": 400,
  "chance": 6,
  "real_chance": 6
}
```

Ссылка `nft_link` не показывается пользователю и не возвращается в публичных данных кейса. Для анимированного содержимого кейса фон принудительно равен `#242424`, узор полностью скрыт. Если `nft_link` отсутствует, показывается `image_url`.

## 6. Контрольная проверка

1. В каталоге прокрутите NFT в видимую область: анимация должна проиграться один раз и замереть.
2. Наведите мышь на ещё не проигранную карточку: анимация должна проиграться один раз.
3. Откройте кейс с `nft_link`: модель должна двигаться на сером фоне без узора.
4. Поставьте 10, затем ещё 200 звёзд в одном раунде: история должна показать одно списание `−210`.
5. После проигрыша дополнительной строки быть не должно.
6. После выигрыша должна появиться отдельная положительная строка с начислением.
7. Проверьте реферальный платёж: начисляется 10%, дробная часть отбрасывается. От 99 звёзд начислится 9.
8. Откройте историю инвентаря: разные операции должны иметь разные премиальные иконки.

## 7. Откат

Если Render показывает ошибку, откройте **Manual Deploy → Deploy a specific commit** и разверните предыдущий commit. Если ошибка относится к миграции данных, временно используйте сохранённую ветку Neon `before-v8-5`.

Расширенная версия этой же инструкции находится в файле `ИНСТРУКЦИЯ_V8.5.md`.
