DNX STORE V8.9 OPT.8 — УСТАНОВКА

1. Распакуйте архив dnx-store-stars-final-v8.9-opt.8-modal-case-fix.zip.
2. Откройте GitHub-репозиторий проекта и нажмите Add file → Upload files.
3. Перетащите ВСЕ файлы из распакованной папки с заменой существующих.
4. Нажмите Commit changes.
5. Откройте Render → ваш Web Service → Manual Deploy → Deploy latest commit.
6. Дождитесь статуса Live.
7. Полностью закройте мини-приложение Telegram и откройте заново. Если осталась старая версия, очистите кэш Telegram или добавьте к URL мини-приложения параметр ?v=8.9-opt.8.

ЧТО ИСПРАВЛЕНО

1. Каталог: окно NFT сразу показывает только кнопку покупки и кнопку корзины. Кнопки вывода и продажи больше не появляются даже на долю секунды.
2. Инвентарь: окно NFT никогда не показывает кнопку покупки или корзины. Остаются только вывод и продажа/«Нельзя продать».
3. Режим модального окна задаётся до показа и не меняется из-за задержанного ответа базы.
4. Кейс: нулевой кадр анимированного NFT и кадры рулетки используют тот же масштаб 70/30, что и полноценная анимация.
5. Обычные статичные предметы кейса не уменьшались и продолжают отображаться как раньше.
6. Игровой код и механика игры не изменялись.

ПРОВЕРКА ПОСЛЕ ДЕПЛОЯ

1. Откройте подряд несколько товаров каталога: видны только «Купить» и корзина, без промежуточных лишних кнопок.
2. Откройте подряд несколько NFT в профиле: кнопки «Купить» и корзина отсутствуют с первого кадра.
3. Откройте кейс при медленном интернете: нулевой кадр модели занимает около 70% карточки и визуально совпадает по масштабу с загруженной анимацией.
4. Запустите рулетку: анимированные награды в нулевом кадре имеют такой же масштаб, статичные награды остались прежнего размера.

ВАЖНО

- Не загружайте ZIP одним файлом: на GitHub должны быть заменены файлы проекта внутри архива.
- Environment-переменные Render менять не требуется.
- После деплоя сервер и клиент должны отдавать api_release 8.9-opt.8.
