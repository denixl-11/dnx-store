DNX STORE V6.3 — ЧТО ИСПРАВЛЕНО
=================================

1. НИЖНЯЯ ПАНЕЛЬ
- Убран зазор между нижней панелью и основной страницей.
- Область под панелью до физического края экрана закрыта собственной подложкой.
- Розовая нижняя линия и скруглённые углы остаются полностью видимыми.
- Геометрия проверена на ширине 360 px, 390 px и на Desktop.

2. ЖИВЫЕ NFT
- Фильтр больше не удаляет и не создаёт заново карточки каталога.
- При уходе со страницы TGS-анимация ставится на паузу, а при возвращении продолжается.
- Переключение «Каталог / Кейсы» больше не уничтожает NFT-плееры.
- Повторные открытия каталога используют кратковременный кэш и не создают лишние запросы.
- В модальном окне NFT используется тот же живой TGS-превью-блок.

3. УЗОР TELEGRAM NFT
- Вместо повторяющейся сетки показывается 5–8 отдельных символов.
- Позиции и поворот распределены детерминированно для каждого NFT.
- Символы разнесены по заранее безопасным зонам и не склеиваются друг с другом.

4. КЕЙСЫ
- Подложки предметов в рулетке и списке дропов получили скругление.
- Добавлен глубокий фиолетово-малиновый градиент и мягкое переливание.
- Сами исходные изображения предметов не изменялись.

5. ANDROID И ПРОИЗВОДИТЕЛЬНОСТЬ
- На Android одновременно используются не более двух TGS-worker вместо четырёх.
- Снижено разрешение внутреннего canvas TGS без изменения его видимого размера.
- Отключены тяжёлые backdrop-filter и декоративные бесконечные эффекты.
- AudioContext создаётся только в момент реального звука кейса.
- Скрытая игровая анимация ожидания не работает в других вкладках.
- Опрос игры вне вкладки «Игра» выполняется реже; при входе обновление идёт сразу.
- Высота приложения берётся из актуального Android viewport, чтобы не появлялся чёрный экран.

ВАЖНО
- main.py, база данных, ставки, траектория мяча, расчёты и определение победителя не менялись.
- Environment-переменные и миграции базы данных для V6.3 не нужны.


КАК УСТАНОВИТЬ V6.3 ЧЕРЕЗ GITHUB
================================

Вариант A — заменить файлы через сайт GitHub

1. Распакуйте архив dnx-store-stars-final-v6.3.zip.
2. Откройте в браузере репозиторий:
   https://github.com/denixl-11/dnx-store
3. Нажмите кнопку «Add file».
4. Выберите «Upload files».
5. Перетащите ВЕСЬ содержимый набор файлов из папки
   dnx-store-stars-final-v6.3 в область загрузки GitHub.
   Перетаскивать надо сами файлы, а не внешнюю папку.
6. Убедитесь, что среди загружаемых файлов есть:
   index.html
   main.py
   tgsticker.js
   tgsticker-worker.js
   rlottie-wasm.js
   rlottie-wasm.wasm
   pako-inflate.min.js
   requirements.txt
   cosmic-bg.png
   game-waiting-bg.png
   history-empty-bg.png
7. Внизу страницы в поле Commit message напишите:
   DNX Store V6.3
8. Выберите «Commit directly to the main branch».
9. Нажмите «Commit changes».

Если GitHub не предлагает заменить существующие файлы:
1. Откройте index.html в репозитории.
2. Нажмите значок карандаша «Edit this file».
3. Полностью замените содержимое кодом из нового index.html.
4. Нажмите «Commit changes».
5. Для остальных изменившихся файлов повторите то же действие.


ПРОВЕРКА RENDER
===============

1. Откройте https://dashboard.render.com
2. Войдите в аккаунт.
3. Откройте Web Service, который обслуживает DNX Store.
4. Перейдите на вкладку «Events».
5. После коммита GitHub должен появиться новый Deploy.
6. Дождитесь зелёной надписи «Deploy live».
7. Если автоматический Deploy не начался:
   - нажмите «Manual Deploy»;
   - выберите «Deploy latest commit»;
   - дождитесь «Deploy live».
8. Environment менять не надо.


ОБНОВЛЕНИЕ TELEGRAM MINI APP
============================

Если URL приложения не изменился, в BotFather ничего менять не надо.

После зелёного Deploy:
1. Полностью закройте мини-приложение Telegram.
2. Закройте чат с ботом.
3. Откройте чат заново.
4. Запустите мини-приложение.
5. Если видна старая версия, подождите 1–2 минуты и повторите закрытие/открытие.


ЧТО ПРОВЕРИТЬ ПОСЛЕ DEPLOY
==========================

1. Откройте каталог на Android и iPhone.
2. Примените по очереди несколько фильтров и сбросьте фильтр.
3. Перейдите в «Кейсы», затем снова в «Каталог».
4. Убедитесь, что NFT продолжают двигаться и не исчезают.
5. Нажмите на NFT и проверьте живой квадрат в модальном окне.
6. Проверьте, что в фоне видно только 5–8 разнесённых символов.
7. Прокрутите каталог до самого низа: под навигацией не должно быть карточек или пустой полосы.
8. Откройте кейс и проверьте скруглённые фиолетовые подложки рулетки и дропов.
9. На Android несколько раз переключите все три нижние вкладки: чёрного экрана быть не должно.

Если после Deploy Telegram показывает старый index.html, очистите кэш Telegram либо временно
добавьте к URL Mini App параметр версии, например ?v=63, а после обновления верните обычный URL.
