# DNX Store V8.5 — установка и проверка

Эта версия подготовлена поверх V8.4. Новые переменные Environment не нужны.

## Что изменено

- Заголовок открытого кейса поднят и выровнен с кнопкой «Назад».
- История инвентаря получила отдельные премиальные SVG-иконки для пополнений, покупок, ставок, выигрышей, кейсов, продаж и вывода.
- Несколько ставок одного пользователя в одном раунде объединяются в одну строку истории. Например, `10 + 200` отображается как одно списание `−210`.
- Для проигравшего больше не создаётся строка «Раунд окончен без выигрыша». В истории остаётся только ставка; у победителя дополнительно появляется начисление выигрыша.
- Реферальное начисление увеличено до 10% и всегда округляется вниз до целой звезды.
- Основной и бонусный балансы, цены, ставки, продажи, пополнения и выигрыши приводятся к целым звёздам округлением вниз.
- В таблице `items` характеристики перенесены из старого JSON-поля `traits` в три обычных столбца: `model`, `pattern`, `background`.
- Анимация NFT в каталоге проигрывается один раз при появлении карточки в области просмотра или при наведении, затем останавливается. Размер модели уменьшен примерно до 65% площади.
- Для содержимого кейса поддерживается `nft_link`. Ссылка не передаётся клиенту; фон анимации всегда `#242424`, узор скрыт. Без `nft_link` остаётся обычная картинка.

## Шаг 1. Сделайте резервную копию Neon

1. Откройте сайт Neon и войдите в свой проект.
2. Откройте раздел **Branches**.
3. Нажмите **Create branch**.
4. Назовите ветку, например `before-v8-5`.
5. Дождитесь создания ветки и только после этого продолжайте.

Это важно: при первом запуске V8.5 сервер автоматически переносит данные из `items.traits` в новые столбцы и удаляет старый столбец `traits`.

## Шаг 2. Замените файлы в GitHub

1. Откройте репозиторий `denixl-11/dnx-store` на GitHub.
2. Нажмите **Add file → Upload files**.
3. Перетащите все файлы из папки `dnx-store-stars-final-v8.5`.
4. Если GitHub спросит о замене одноимённых файлов, подтвердите замену.
5. Внизу страницы напишите сообщение коммита, например `DNX Store V8.5`.
6. Нажмите **Commit changes**.

Обязательно должны быть загружены как минимум:

- `main.py`
- `index.html`
- `tgsticker.js`
- `tgsticker-worker.js`
- `rlottie-wasm.js`
- `rlottie-wasm.wasm`
- `pako-inflate.min.js`
- все PNG/WEBP-файлы из комплекта

## Шаг 3. Проверьте Render

1. Откройте Render.
2. Выберите Web Service вашего бота.
3. Откройте **Settings** и проверьте:
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python main.py`
   - Health Check Path: `/health`
4. Откройте **Environment**. Старые значения оставьте без изменений; новые переменные не добавляются.
5. Откройте **Manual Deploy**.
6. Нажмите **Deploy latest commit**.
7. Откройте **Logs** и дождитесь сообщения об успешном запуске. Первый запуск может занять немного больше времени из-за миграции базы.

## Шаг 4. Как заполнять товары каталога

После первого запуска в `items` должны быть столбцы:

- `model`
- `pattern`
- `background`

Если характеристика отсутствует, оставьте значение `NULL`. PostgreSQL хранит имена этих столбцов в нижнем регистре — это нормально.

## Шаг 5. Как добавить анимированный предмет в кейс

В переменной `CASES_JSON` у нужного элемента `drops` добавьте `nft_link`. Пример:

```json
{
  "name": "Chill Flame",
  "image_url": "https://example.com/fallback.png",
  "nft_link": "https://t.me/nft/ChillFlame-12345",
  "model": "Bear Market",
  "pattern": "Oculus",
  "background": "Chocolate",
  "value": 400,
  "chance": 6,
  "real_chance": 6
}
```

Внутри кейса `pattern` и исходный цвет фона намеренно не показываются: интерфейс использует ровный фон `rgb(36, 36, 36)` / `#242424`. Поле `nft_link` не возвращается в публичном JSON API. Если ссылки нет, используется `image_url`.

## Шаг 6. Проверка после обновления

1. Откройте каталог на iPhone, Android и компьютере.
2. Прокрутите карточку NFT в область просмотра: анимация должна проиграться один раз и остановиться.
3. Уведите карточку с экрана и верните обратно: она может проиграться ещё один раз при новом появлении.
4. Откройте кейс с `nft_link`: модель должна двигаться, фон должен быть `#242424`, узора быть не должно.
5. Сделайте две ставки в одном раунде, например 10 и 200 звёзд. В истории должна быть одна строка `−210`.
6. Проиграйте раунд: дополнительной строки о проигрыше быть не должно.
7. Выиграйте раунд: должны быть строка общей ставки и отдельная строка начисленного выигрыша.
8. Проверьте реферальное пополнение: владельцу ссылки начисляется 10%, дробная часть отбрасывается. Например, от 99 звёзд начисляется 9.
9. Откройте историю инвентаря и проверьте разные типы операций — у каждой должна быть своя премиальная иконка.

## Если после обновления возникла ошибка

1. Не изменяйте данные вручную наугад.
2. Скопируйте последние строки из Render Logs.
3. При необходимости временно разверните предыдущий GitHub commit через **Manual Deploy → Deploy a specific commit**.
4. Если проблема связана с миграцией, переключитесь на созданную перед обновлением ветку Neon `before-v8-5`.
