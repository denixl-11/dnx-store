# DNX Store V8.9 — характеристики, база и интерфейс

## Почему возвращались старые характеристики

В предыдущем сервере оставался аварийный способ подключения: если в Render не было `DATABASE_URL`, приложение брало только `DB_PASSWORD` и подключалось к жёстко записанному старому Neon-host. Из-за этого вы могли менять `items` в одной базе/ветке Neon, а Render читал другую.

В V8.9 этот небезопасный fallback удалён. Сервер принимает только:

- `DATABASE_URL` — рекомендуемый вариант; или
- полный комплект `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_PORT`.

Модалка получает `model`, `pattern`, `background` отдельным запросом по точному `items.id`. Поле `items.traits` не используется.

## Обновление

1. Загрузите в GitHub всё содержимое папки `dnx-store-stars-final-v8.9` с заменой файлов.
2. В Render откройте сервис бота → **Environment**.
3. В Neon откройте нужный проект и нужную branch → **Connect** → скопируйте pooled connection string.
4. В Render создайте или замените переменную `DATABASE_URL` этим значением. Не отправляйте строку подключения другим людям.
5. Нажмите **Save Changes**, затем **Manual Deploy** → **Deploy latest commit**.
6. В логах Render найдите строку `DNX database audit`. В ней должны быть release `8.9`, имя базы, схема, fingerprint и столбцы `model,pattern,background`.
7. После деплоя откройте `https://dnx-shop.onrender.com/health`. Ответ должен содержать `"release":"8.9"`.
8. Полностью закройте мини-приложение Telegram и откройте заново.

## Проверка конкретного NFT

Администратор может написать боту:

`/db_audit ID`

Например: `/db_audit 42`.

Бот покажет точные `model`, `pattern`, `background` строки, которую читает Render. Если в правильной строке записано `kall`, команда и модалка покажут `kall`. Если команда показывает старые значения, изменена другая база, другая branch или другая строка `items.id`.

## Остальные исправления

- Нижняя вкладка `ИНВЕНТАРЬ` переименована в `ПРОФИЛЬ`.
- Внутренняя вкладка `МОИ NFT` переименована в `ИНВЕНТАРЬ`.
- Кнопки вывода и продажи имеют одинаковую ширину и высоту.
- Цена продажи отделена от текста; звезда больше не слипается с числом.
- `НЕЛЬЗЯ ПРОДАТЬ` снова серая.
- Стрелка кейса и название находятся на одной горизонтали.
- Метка `ДОСТУПНО ЗА БОНУСЫ` уменьшена.
- Освобождается лишняя копия кадров TGS; сохранённый медиакэш уменьшен для снижения памяти и лагов WebView.
