DNX STORE V6.4 — УСТАНОВКА И ПРОВЕРКА
======================================

ЧТО ИЗМЕНЕНО
-------------
1. Анимация Telegram NFT подготавливается сервером заранее и запускается без прежней паузы.
2. Анимированный NFT теперь используется в каталоге, корзине, инвентаре и окне характеристик.
3. Метаданные анимации и каталог безопасно кешируются; после фильтра, смены вкладки и возврата анимация снова запускается.
4. В кейсах исправлены внутренние скругления, размеры рулетки на macOS и серо-фиолетовое переливание фона.
5. Вместо ромба TON установлен логотип Toncoin.
6. Добавлено автоматическое восстановление Telegram Stars-платежей, если Render пропустил update successful_payment.
7. Добавлены две административные команды: /stars_balance и /stars_reconcile.

ВАЖНО ПРО TELEGRAM STARS
------------------------
Оплата Stars поступает не на личный пользовательский баланс администратора. Она поступает на отдельный доходный баланс Telegram-бота.

Одновременно DNX Store должен увеличить внутренний баланс пользователя на ту же сумму. В V6.4 это происходит двумя способами:
- сразу по событию successful_payment;
- автоматически по официальной истории операций бота, если событие было пропущено во время перезапуска Render.

Один платёж нельзя начислить дважды: используются уникальные invoice_payload и Telegram transaction id, а запись обновляется внутри транзакции базы данных.

КАК ПРОВЕРИТЬ ТЕСТОВЫЕ 10 STARS
-------------------------------
1. Сначала разверните V6.4 по инструкции ниже.
2. Откройте обычный чат со своим ботом в Telegram.
3. Отправьте боту команду:
   /stars_balance
4. Бот покажет свой доходный Stars-баланс и последние операции. Команда доступна только Telegram ID из переменной ADMIN_ID.
5. Затем отправьте:
   /stars_reconcile
6. Команда сверит последние 100 операций Telegram с таблицей star_payments и восстановит пропущенное внутреннее зачисление.
7. Закройте мини-приложение, откройте заново и проверьте баланс.

Если /stars_balance не отвечает:
- проверьте, что ваш личный Telegram ID точно записан в Render -> Environment -> ADMIN_ID;
- проверьте BOT_TOKEN;
- откройте Render -> Logs и найдите строку "Admin Stars balance command error".

КАК УСТАНОВИТЬ V6.4 ЧЕРЕЗ GITHUB В БРАУЗЕРЕ
-------------------------------------------
1. Распакуйте архив dnx-store-stars-final-v6.4.zip.
2. Откройте https://github.com/denixl-11/dnx-store.
3. Нажмите Add file.
4. Нажмите Upload files.
5. Откройте распакованную папку dnx-store-stars-final-v6.4.
6. Выделите ВСЕ файлы внутри папки, а не саму папку.
7. Перетащите файлы на страницу GitHub.
8. В поле Commit changes напишите: DNX Store V6.4.
9. Нажмите Commit changes.
10. Не меняйте Environment в Render: новых обязательных переменных нет.

КАК ПРИМЕНИТЬ НА RENDER
-----------------------
1. Откройте https://dashboard.render.com.
2. Выберите Web Service вашего DNX Store.
3. Откройте вкладку Events и дождитесь автоматического Deploy после GitHub commit.
4. Если Deploy не начался, нажмите Manual Deploy -> Deploy latest commit.
5. Откройте Logs.
6. Дождитесь запуска без traceback и появления сообщения об инициализации базы.
7. Первый старт может занять до нескольких секунд дольше: сервер заранее прогревает анимации первых NFT.

ЧТО ПРОВЕРИТЬ ПОСЛЕ DEPLOY
--------------------------
1. Полностью закройте мини-приложение Telegram и откройте его снова.
2. Откройте Магазин -> Каталог: первые NFT должны начать анимацию сразу после отрисовки вкладки.
3. Примените фильтр, сбросьте его, перейдите в Кейсы и вернитесь: анимация должна продолжиться.
4. Добавьте NFT в корзину: внутри строки корзины должен быть маленький анимированный квадрат.
5. Откройте Инвентарь: предметы с Telegram nft_link должны быть анимированы.
6. Откройте кейс на Android и macOS: внутренние карточки должны иметь гладкие скругления, без серых острых квадратов и без разъезда размеров.
7. Откройте Пополнение: возле Toncoin должен быть синий логотип TON.
8. Проведите тестовое пополнение минимальной суммой и проверьте сообщение в чате, всплывающее окно и внутренний баланс.
