DNX STORE V8.1 — КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ ИНВЕНТАРЯ
==================================================

Эту версию нужно загружать целиком поверх V8.

Исправлено:
1. /inventory снова корректно возвращает данные: поля времени преобразуются в
   ISO-строки и больше не ломают JSON-ответ.
2. После успешной продажи кейсовый NFT немедленно исчезает из активного
   инвентаря, даже если последующее обновление интерфейса задержалось.
3. Проданный NFT остаётся в таблице items со статусом disposed и отображается
   в истории как ПРОДАНО.
4. Продажа и вывод одного предмета защищены серверными атомарными UPDATE:
   успешно выполнится только одно действие.
5. На время запроса продажи или вывода обе кнопки предмета блокируются.
6. После запроса вывода сразу появляется серый таймер 02:00:00. Таймер
   сохраняется после перезапуска Mini App и обновляется каждую секунду.
7. Все значки основной и бонусной звезды имеют тот же вычисленный размер
   шрифта, что и соседние цифры, включая MacBook, шапку, баланс, пополнение,
   ставки, цены и историю.
8. Золотая рамка победителя показывается 2 секунды — на 0.5 секунды дольше,
   чем в V8 — до появления окна победителя.

Установка:
1. Сделайте резервную ветку Neon и резервную копию GitHub.
2. Откройте GitHub -> denixl-11/dnx-store -> Add file -> Upload files.
3. Перетащите ВСЕ файлы из папки dnx-store-stars-final-v8.1 в корень репозитория.
4. Подтвердите замену index.html и main.py.
5. Commit message: DNX Store V8.1 inventory hotfix.
6. Render -> Manual Deploy -> Clear build cache & deploy.
7. Дождитесь успешного /health.

Новых Environment-переменных и ручного SQL не требуется.
